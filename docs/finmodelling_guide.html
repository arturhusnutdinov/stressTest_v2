Главная идея — в трёх предложениях

Зачем нужна 3-statement модель

Финансовая модель «трёх отчётов» — это связанный прогноз Отчёта о прибылях и убытках (IS), Баланса (BS) и Отчёта о движении денежных средств (CF) компании на 5 лет вперёд. Модель учитывает взаимозависимости: процентные расходы зависят от долга, долг — от свободного денежного потока, а денежный поток — от чистой прибыли, которая уже включает проценты.

Движок решает эту циклическую зависимость итерационно: для каждого года прогоняет цикл «Долг → Проценты → Налог → Прибыль → Cash → Долг» до сходимости (tolerance $1,000, max 10 итераций). На выходе — полностью сбалансированный прогноз, где Активы = Пассивы + Капитал с точностью до $0.000004.

Поверх базового прогноза работают три аналитических модуля: стресс-тестирование (8 сценариев для Rusal), кредитный рейтинг (методология S&P) и мониторинг ковенантов (breach/warning/ok).

04

Итеративный солвер — циклическая зависимость

Долг зависит от Cash, Cash — от NI, NI — от процентов, проценты — от долга. Решается итерационно.

Почему нужны итерации

В 3-statement модели есть circular reference: процентные расходы зависят от суммы долга, а сумма долга зависит от свободного денежного потока, который включает процентные расходы. В Excel это решается «enable iterative calculation». У нас — явный цикл.

Для каждого прогнозного года: сначала рассчитываются «однопроходные» блоки (Revenue, COGS, SGA, PPE, WC, Leases), затем запускается итерационный цикл с 8 блоками, пока не достигнута сходимость.

// Однопроходные блоки:
Revenue → COGS → SGA → PPE(D&A) → WC → Leases → Intangibles

// Итерационный цикл (max 10, tol $1,000):
loop {
  Debt(interest) → Interest_Payable → IS_Subtotals(EBITDA→EBT)
  → Tax(NOL) → Equity(div, buyback) → CF(CFO+CFI+CFF)
  → Cash → BS_Totals → Covenant_Acceleration_Check
} until |Δcash| < $1,000 and |Δni| < $1,000

Долговой движок — instrument-level corkscrew

DebtOptimizer (581 строка): каждый из 70 инструментов моделируется отдельно. ST/LT классификация по 4 правилам.

Как работает DebtOptimizer

Для каждого прогнозного года: (1) определяются обязательные погашения по каждому инструменту, (2) истекшие инструменты рефинансируются, (3) при нехватке Cash — draw из RC (если есть), (4) при избытке Cash — добровольное погашение, (5) начисляются проценты (fixed: по ставке, floating: base_rate + spread), (6) ST/LT split, (7) CFF агрегация.

ST/LT классификация (4 правила в порядке приоритета)

Quick Start

from engine.orchestrator import build_model

# Базовая модель
result = build_model("rusal", run_model=True)

# Полный pipeline
result = build_model("rusal",
    run_model=True,
    run_stress=True,    # 8 сценариев
    run_rating=True,    # S&P scorecard
    run_covenants=True, # breach monitoring
)
print(f"BS = {max(result.model_result.bs_diffs.values()):.6f}")
print(f"Stress: {len(result.stress_results)} scenarios")
