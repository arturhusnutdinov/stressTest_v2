Architecture — Layered Pipeline

Data flows through 6 stages. Each stage reads from the previous, writes to SQLite. No circular dependencies between stages.

Preprocessor14 metric groups
→
Macro ModuleVECM / ARIMA / ECM
→
Model LoaderHistoricState + Config
3-Statement SolverJoint iteration
→
Stress RunnerScenario shocks
→
Rating + CovenantsS&P / Moody's / Fitch
Design principle: Model does not know about DB or YAML. All data arrives as typed dataclasses (ModelConfig, HistoricState, YearState). The loader is the only place that touches files and DB.

Data Model — YearState

The central dataclass. One instance per forecast year. 80+ fields covering full IS, BS, and CF.

StatementKey FieldsValidation
Income Statement revenue, cogs, sga, ebitda, ebit, interest_expense, ebt, tax_expense, net_income EBITDA = GP + SGA − DA
EBT = EBIT + non-op items
Balance Sheet cash, ar, inventory, ppe_net, rou_asset, intangibles, st_debt, ap, lt_debt, equity A = L + E (tol $1)
Cash Flow cfo_total, cfi_total, cff_total, cf_cash_ending Cash_end = Cash_open + CFO + CFI + CFF (tol $1)
# BS Identity check — called after every forecast year
total_assets = CA + NCA
total_liab_equity = CL + NCL + Equity
diff = |total_assets − total_liab_equity| < $1

Every line item in IS/BS/CF can use a different method. Configured in project.yaml → forecast_methods.

MethodDescriptionUsed For
MACROElastic Net regression on macro factors (GDP, CPI, PPI, commodity prices)Revenue, COGS
DRIVER% of base article (e.g. COGS = 72% × Revenue)COGS, SGA

06

Debt Engine — 3 Modes

The debt solver supports full instrument-level modeling with refinancing, floating rates, and covenant acceleration.

ModeDescriptionWhen to Use
optimizer Full LP optimization: 7-step algorithm with refinancing, maturity rollover, voluntary prepay caps, ST/LT split by 4 rules Detailed instrument data available (US Steel: 38 instruments)
schedule_based Reads pre-built corkscrew from debt_schedule table Analyst has exact repayment schedule
parametric Simplified: target ND/EBITDA or Debt/Revenue ratio, ST/LT split from history Limited instrument data; quick estimates
Floating rate support: Instruments with rate_type=floating receive general_rate_delta_pct shocks. CBR key rate forecast passed per-year. Fixed-rate instruments unaffected by rate stress.

Adding a New Company

# 1. Create company structure
python3 tools/init_company.py my_company --name "My Company" --currency USD

# 2. Fill Excel template with historical data
#    Copy templates/excel_templates/template_UNIFIED_All_Data.xlsx
#    Fill IS/BS/CF sheets with 5+ years of data

# 3. Load into database
from engine.loader.excel import ExcelLoader
from engine.database.repository import Repository
with Repository() as repo:
    loader = ExcelLoader("my_company", repo, db_unit="USD")
    loader.load(Path("companies/my_company/data/my_company.xlsx"))

# 4. Run model
result = build_model("my_company")
