# 📓 Универсальные ноутбуки

Эти ноутбуки предназначены для работы с **любой компанией** из папки `companies/`.

## 📋 Доступные ноутбуки

### `00_Build_Model_Main.ipynb`
**Главный ноутбук построения модели**

Позволяет:
- Выбрать компанию и режим модели
- Просмотреть канонические формы отчетности
- Просмотреть все настройки из YAML
- Запустить полный pipeline (макро-прогноз → модель → валидация → стресс → рейтинг)
- Создать новую модель для новой компании

### `01_Test_Macro_Module.ipynb`
**Тестирование модуля макропрогнозирования**

Поэтапное тестирование:
1. Загрузка конфигурации
2. Проверка поиска файлов
3. Чтение макро-факторов
4. Анализ данных
5. Тест VECM модуля
6. Запуск VECM прогнозирования
7. Проверка выходных данных

### `99_Configure_YAML.ipynb`
**Интерактивная настройка YAML конфигурации**

Позволяет:
- Настраивать параметры через виджеты
- Валидировать конфигурацию
- Просматривать изменения перед сохранением

## 🚀 Использование

### Способ 1: Прямое использование универсального ноутбука

1. Скопируйте нужный ноутбук в рабочую директорию:
   ```bash
   cp templates/notebooks/00_Build_Model_Main.ipynb ./00_Build_Model_Main.ipynb
   ```

2. Откройте ноутбук:
   ```bash
   jupyter notebook 00_Build_Model_Main.ipynb
   ```

3. В разделе "Импорты и настройка" укажите компанию:
   ```python
   COMPANY = "your_company_name"  # Измените на имя вашей компании
   ```

### Способ 2: Использование через companies/{company}/notebooks/

1. Создайте символическую ссылку или скопируйте ноутбук:
   ```bash
   mkdir -p companies/my_company/notebooks
   cp templates/notebooks/00_Build_Model_Main.ipynb companies/my_company/notebooks/
   ```

2. Откройте ноутбук из папки компании

## 📝 Настройка COMPANY

В каждом ноутбуке есть переменная `COMPANY`, которую нужно установить:

```python
# Настройка пути к корню проекта
ROOT = Path('.').resolve()  # или Path('..').resolve() если ноутбук в подпапке

# ===== ВАЖНО: ИЗМЕНИТЕ ЭТУ ПЕРЕМЕННУЮ =====
COMPANY = "your_company_name"  # ← Измените на имя вашей компании

# Проверка существования компании
croot = ROOT / "companies" / COMPANY
if not croot.exists():
    print(f"❌ Компания '{COMPANY}' не найдена в {croot}")
    print(f"Доступные компании: {[d.name for d in (ROOT / 'companies').iterdir() if d.is_dir()]}")
```

## ✅ Требования

1. **Структура компании должна существовать:**
   ```
   companies/{COMPANY}/
   ├── configs/
   │   └── project.yaml
   ├── history/
   │   ├── is_history_{COMPANY}.csv
   │   ├── bs_history_{COMPANY}.csv
   │   └── cf_history_{COMPANY}.csv
   └── ...
   ```

2. **Конфигурация должна быть настроена:**
   - `companies/{COMPANY}/configs/project.yaml` должен существовать
   - Макро-факторы должны быть указаны
   - Файлы макро-факторов должны быть доступны

## 🔄 Обновление ноутбуков

При обновлении универсальных ноутбуков в `templates/notebooks/`:
- Компания-специфичные копии в `companies/{company}/notebooks/` НЕ обновляются автоматически
- Пользователи должны вручную обновить свои копии
